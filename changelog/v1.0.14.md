# RxFHIR v1.0.14

## Added

- Added a full accessibility settings area with motion preference, text scale, interface zoom, and enhanced focus visibility controls.
- Added accessibility planning, checklist, component rules, and manual test documentation under `docs/accessibility/`.
- Added `npm run a11y:check` as a repository smoke check for core accessibility structure and locale coverage.

## Changed

- Expanded keyboard and screen-reader support across App Shell, Creator, Consumer, Settings, dialogs, shortcut editing, and technical inspectors.
- Improved JSON Viewer and FHIR Request Inspector with readable summary mode alongside raw JSON.
- Updated README to document the current accessibility capabilities, smoke check workflow, and release expectations.

## Fixed

- Fixed version drift between package metadata, README, About content, and the macOS custom About window.
- Fixed high-zoom layout behavior in Creator and Consumer by allowing key panels to reflow vertically on narrower effective viewports.
- Fixed shared UI states so disabled, selected, and focus-visible states remain more readable in high-contrast and low-vision scenarios.
