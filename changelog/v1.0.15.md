# RxFHIR v1.0.15

## Added

- Added a broader UX layer across the desktop app, including Activity Center, Command Palette, first-run onboarding, task-based Quick Start scenarios, and Creator leave-guard workflows.
- Added Electron-native preference import/export, recent local bundle file handling, drag-and-drop Bundle inspection, and persisted window state behavior.
- Added repo-native Electron UX verification tooling, including `npm run cdp:eval`, `npm run ux:electron:smoke`, `npm run ux:electron:verify`, and supporting documentation under `docs/ux/`.

## Changed

- Updated README to reflect the current UX capabilities, Electron debug workflow, and release verification expectations.
- Improved Creator and Consumer workflows with clearer draft state, post-submission diff summaries, richer toast-to-activity feedback, and a more complete desktop query workspace.
- Refined the title-bar utility controls with better spacing consistency and a cleaner unread badge presentation for Activity Center.

## Fixed

- Fixed the Electron startup blocker caused by inherited `ELECTRON_RUN_AS_NODE` state by routing repo scripts through a clean Electron launcher.
- Fixed native Bundle JSON and Preferences JSON bridge flows so open/save dialog testing is now verified in the real Electron window.
- Fixed version drift so package metadata, README, Settings About content, and the custom macOS About window all align on `1.0.15`.
