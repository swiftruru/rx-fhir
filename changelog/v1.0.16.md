# RxFHIR v1.0.16

## Added

- Added Live Demo visibility choreography so the demo now scrolls the stepper, form fields, and submit actions into view before interacting with them.
- Added split-view resizing for the Creator info panel in both side-by-side and bottom-panel layouts, with persisted panel size across launches.
- Added post-submit Live Demo request showcase behavior that temporarily collapses the teaching coach, highlights the FHIR request panel, and then resumes the guided lesson.

## Changed

- Updated Quick Start and onboarding flows so the blank Creator entry no longer auto-loads sample data, reducing confusion with placeholder text during first use.
- Refined Creator, Consumer, Settings, Activity Center, Command Palette, and onboarding visuals for a more consistent desktop UX rhythm.
- Changed the Creator inspector wording from a location-specific “right panel” label to the neutral “info panel” label.
- Defaulted FHIR request and response body inspection to the raw view for a more technical, professional presentation.

## Fixed

- Fixed Live Demo wording that incorrectly referenced `MedicationRequest.author`; the narrative now correctly uses `MedicationRequest.requester`.
- Fixed Creator narrow-width layout regressions where the header, stepper, or info panel could crowd the form area or break expected scrolling behavior.
- Fixed version drift so package metadata, README, Settings About content, and the custom macOS About window all align on `1.0.16`.
