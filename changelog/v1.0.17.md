# RxFHIR v1.0.17

## Added

- Added Export dropdown in Consumer prescription detail with three formats: FHIR JSON, Postman Collection v2.1, and HTML report.
- Added Postman Collection export that probes the FHIR server for each resource to determine PUT vs POST automatically, using HAPI-2840 duplicate detection and identifier/code search fallback.
- Added sequential probing (one resource at a time) to prevent 429 rate limiting on public HAPI servers.
- Added four query request items in the exported Postman collection covering basic, date, and complex search modes, each with a client-side filter test script that mirrors the app's own workaround logic.
- Added HTML report export that generates a self-contained, printable prescription summary with embedded CSS.

## Fixed

- Fixed date search now correctly filtering by Composition.date (the prescription date) instead of Bundle.timestamp (the server submission time).
- Fixed date query example pre-filling with the actual Composition.date from the most recently submitted bundle instead of a dynamic relative date.
- Fixed Feature Showcase polluting the Activity Center by clearing notification history on showcase start and restoring it on exit.
