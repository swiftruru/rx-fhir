# RxFHIR v1.0.19

## Added

### Creator — FHIR Request Inspector Postman Export
- Added a one-click **Export Postman** button in the FHIR Request Inspector panel header; the button is enabled whenever the request history contains at least one entry.
- Generates a Postman Collection v2.1 from the captured session history, including POST resource creation, PUT updates, and GET check/search requests.
- Collection uses a `{{fhirBaseUrl}}` variable for portability; query parameters are decoded for human readability in Postman's UI.
- Export is fully synchronous — no server probing required, since method and URL are already known from the inspector history.
- Added `stepper.exportPostman` i18n key in both `en` and `zh-TW` creator namespaces.

### Consumer — HTML Report Enhancements
- **Clinical timeline**: key clinical events (encounters, diagnoses, observations, medications, prescriptions) are rendered in chronological order as a visual timeline below the document header.
- **Medication summary table**: a compact table lists all prescribed medications before the detailed resource cards.
- **Lab value indicators**: Observation cards now show color-coded badges (Normal ✓ / High ↑ / Low ↓) derived from `referenceRange[0].low` and `.high` comparisons.
- **Composition status banner**: a color-coded banner at the top of the report reflects `Composition.status` — green for `final`, yellow for `draft`, blue for `amended`, red for `entered-in-error` — and shows the attester name if present.
- **Global full-text search**: a search bar in the top-right corner scans all visible text (excluding raw JSON blocks) using a `TreeWalker`, highlights matches with `<mark>`, and provides ↑↓ navigation with a live `{current}/{total}` count; pressing Escape clears the search.
- **Print layout**: added `@page` A4 margins (18 mm top/bottom, 14 mm sides), a fixed print header with the patient name and document date, and a fixed print footer with the page number placeholder; print controls are hidden when printing.
- **QR code modal fix**: moved the `<dialog id="qr-modal">` to body level so `showModal()` positions it relative to the viewport rather than an ancestor stacking context; dialog now uses `position: fixed; inset: 0; margin: auto` for consistent centering.
- **Global search pill layout**: wrapped the search input and ↑↓ navigation buttons in a shared bordered `.gs-pill` container so the buttons appear visually adjacent to the input field.

### Consumer — Postman Export Cancellation
- Added mid-flight cancellation support for the Consumer Postman export operation.
- While the export is probing the FHIR server, the Export dropdown is replaced by an inline cancel button showing a progress counter (`{checked}/{total}`) and a brief status label.
- Cancellation is handled via `AbortController`; an `AbortError` is caught silently with no error toast.
- Added `detail.exportPostmanChecking` i18n key in both `en` and `zh-TW` consumer namespaces.

## Fixed

- Fixed Creator Postman export generating incorrect URL structure in Postman (domain and base path appearing as part of the path instead of the host variable): `url.host` is now `['{{fhirBaseUrl}}']` and `url.path` contains only the resource path segments.
- Fixed timeline date column overflowing when Chinese locale produced long date+time strings: date-only formatting (`fdateOnly`) is now used for timeline labels, and the left column is widened to 7 rem.
- Fixed global search ↑↓ arrows appearing far from the search input due to `min-width` spacing on the match counter.
