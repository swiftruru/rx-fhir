# RxFHIR v1.0.20

## Added

### About Page Redesign
- Replaced the single-card About layout with a structured Hero + two-card design.
- **Hero block**: app icon (with hover scale effect and primary-tinted ring), app name, tagline, description, and dynamic version number (`v1.0.20`) injected at build time via a Vite `define` constant — no more hardcoded strings.
- **Academic background card**: author, affiliation, department, and course fields with `BookOpen` icon header; advisor badges using shadcn `Badge` component with `GraduationCap` icon section header.
- **Technical info card**: FHIR spec, tech stack summary, GitHub repository link (`swiftruru/rx-fhir`), and personal homepage link (`swift.moe`) — both open in the system browser via the existing `openExternalUrl` bridge.
- MIT license footer at the bottom of the page.
- Both author name fields now show the bilingual form (`潘昱如（Yu-Ru Pan）` / `Yu-Ru Pan（潘昱如）`) so the About page reads naturally in either locale.
- Added `__APP_VERSION__` Vite define in `electron.vite.config.ts` (reads from `package.json` at build time) and a global TypeScript declaration.
- Added PNG asset type declaration (`src/renderer/types/assets.d.ts`) for `import appIcon from '../../assets/icon.png'`.

### Feature Showcase — Export Step
- Added a new **consumerExport** step (step 12 of 16) to the Feature Showcase tour.
- Spotlights the Consumer Export dropdown button with a `glow` highlight; the coaching card appears at `bottom-left` to avoid covering the button in the top-right corner.
- Step UI override keeps the detail panel open in structured view so the export button is always visible.
- Added `consumer.exportButton` targetId and wrapped `ExportDropdown` in `FeatureShowcaseTarget` in `PrescriptionDetail.tsx`.
- Added `consumerExport` step translations in both `zh-TW` and `en` showcase locale files.

## Fixed

### Feature Showcase — Spurious Bundle Toasts
- Starting the Feature Showcase injected a demo `bundleId` into the creator store, which triggered the "Document Bundle created" success toast on every run.
- Fixed by skipping the toast (while still syncing `lastAnnouncedBundleId`) when `featureShowcaseStatus !== 'idle'`, so showcase runs no longer produce spurious Activity Center notifications.

### Electron About Window Version
- The native macOS About window was showing a hardcoded version string `版本 1.0.16`.
- Fixed to use `app.getVersion()` so it always reflects the actual packaged version.

## Changed

- **Creator navigation**: removed the leave-guard confirmation dialog that appeared when navigating away from Creator with unsaved work. Draft auto-save already ensures no data is lost, making the dialog unnecessary friction. Navigation now proceeds immediately.
