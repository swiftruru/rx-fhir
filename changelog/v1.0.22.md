# RxFHIR v1.0.22

## Added

### "Check for Updates…" in macOS Menu Bar
- The **RxFHIR** application menu now includes a **Check for Updates…** item, positioned after "About RxFHIR" — matching the convention used by apps such as VS Code.
- Clicking the item triggers the same GitHub Releases API check introduced in v1.0.21, but runs entirely in the Main Process (no renderer involvement required).
- Three native dialog outcomes:
  - **Up to date** — info dialog confirming the current version is the latest.
  - **Update available** — info dialog showing the new version number, with a **View on GitHub Releases** button that opens the Releases page in the system browser, and a **Later** button to dismiss.
  - **Check failed** — warning dialog asking the user to verify their network connection.
- While the check is running, the menu item label changes to **Checking…** and becomes disabled to prevent duplicate requests. It restores automatically when the result arrives.
- macOS only (the menu bar convention does not apply to Windows or Linux).
