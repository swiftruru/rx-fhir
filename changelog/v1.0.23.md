# RxFHIR v1.0.23

## Fixed

### Update notification now visible from any page
Previously, the startup update check (which runs 5 seconds after launch) would push a result to the renderer, but only the About page had a listener registered. If the About page was not open at that moment, the push was silently lost and the user never saw a notification.

This release moves the update listener to the App shell level so the notification is always received regardless of which page is currently active.

### Changes
- `App.tsx`: registers `onUpdateResult` and `getCachedUpdateResult` at the root `AppShellContent` level; displays a persistent toast (12 s) with a **View on GitHub Releases** action button when a newer version is detected.
- `updateIpc.ts`: caches the startup check result in a module-level variable so pages that mount after the push was sent can still retrieve it via the new `app-update:get-cached` IPC handler.
- `preload.ts` / `electron.d.ts`: expose `getCachedUpdateResult()` on the bridge.
- `AboutPage.tsx`: retains the amber dot indicator and the real-time subscription for updating the dot when the About page is already open; toast responsibility has been moved to App shell to prevent duplicates.
