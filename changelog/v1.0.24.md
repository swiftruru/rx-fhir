# RxFHIR v1.0.24

## Added

### "Skip This Version" update notification option
The update notification now presents two explicit choices instead of a single action:

- **View on GitHub Releases** — opens the release page in the system browser (unchanged behavior).
- **Skip This Version** — permanently suppresses notifications for that specific version number. If a newer release is published later, the notification resumes automatically.

The skipped version is stored in `update-skip.json` inside the app's `userData` directory. Only one version can be skipped at a time; the record is overwritten each time the user chooses to skip.

### Behavior details
- The 5-second startup check reads `update-skip.json` before deciding whether to notify. If `skippedVersion` matches the latest release tag, the check exits silently.
- Clicking **Skip This Version** also clears the in-memory cached result so the amber dot on the About page disappears without a restart.
- The About page update card gains a matching **Skip This Version** ghost button inside the update-available banner. Clicking it calls the same IPC handler and resets the card state to idle.
- The manual **Check for Updates** button on the About page is unaffected — it always returns the live result regardless of the skip setting (because the user explicitly requested the check).

### Technical changes
- `src/main/updater/updateIpc.ts`: adds `readSkippedVersion()` / `writeSkippedVersion()` helpers (following the existing `window-state.json` pattern), extends `scheduleStartupUpdateCheck()` with a skip guard, and registers the new `app-update:skip-version` IPC handler.
- `src/main/preload.ts` / `src/renderer/types/electron.d.ts`: expose `skipUpdateVersion(version)` on the bridge.
- `src/renderer/store/toastStore.ts`: adds `actions?: ToastAction[]` alongside the existing `action?` field for backward compatibility.
- `src/renderer/components/ToastViewport.tsx`: renders all entries in `actions` (or falls back to `action`) as a flex-wrapped row of buttons.
- `src/renderer/App.tsx`: `showUpdateToast` now uses `actions` with both buttons.
- i18n: adds `about.update.skipVersion` key in both `en` and `zh-TW` settings namespaces.
