# RxFHIR v1.0.25

## Added

### "Remind Me Later" option in update notifications

The update notification now offers three choices instead of two:

- **View on GitHub Releases** — opens the release page in the system browser (unchanged behavior).
- **Remind Me Later** — dismisses the current notification without recording anything. The next time the app starts, the 5-second startup check runs as usual and re-prompts if the same (or a newer) version is still available.
- **Skip This Version** — permanently suppresses notifications for that specific version number (unchanged behavior).

The new button appears both in the toast notification and in the About page update banner, positioned between the two existing actions.

### Behavior details

- **Remind Me Later** writes nothing to disk and clears no in-memory state. The amber dot on the About page disappears only for the current session; after a restart it reappears if the version is still newer.
- **Skip This Version** continues to write `update-skip.json` and clear the cached result — behavior is unchanged.
- The three actions are ordered intentionally: primary (go update) → deferral (remind later) → permanent suppression (skip version), so the most consequential action is always last.

### Consumer search improvements

- **Composition chain search** (Step 4): after identifier-based steps, the app queries `GET /Composition?subject=Patient/{id}` and then `GET /Bundle?composition=Composition/{id}` to discover prescriptions submitted from any FHIR client (Postman, third-party apps, etc.), not only those created with RxFHIR's own `Bundle.identifier` convention. Results from all steps are merged and deduplicated by Bundle ID.
- **Cancel search**: an `×` icon button appears beside the submit button while a query is running. Clicking it aborts all in-flight HTTP requests via `AbortController` and displays a distinct cancelled state in the results panel instead of an error.
- **Results cleared on new search**: old results are removed at the start of each new query so stale data cannot mislead the user during the loading period.
- **Loading animation**: while a search is running the results panel shows a spinner and animated skeleton cards.
- **Tab lockout during search**: all three search-mode tabs (Basic / Date / Complex) and the two middle tabs (Results / Shortcuts) are disabled while a query is in progress, preventing accidental tab switches that previously caused the spinner to appear on the wrong tab.
- **Navigation guard**: navigating away from Consumer during an active search now shows an informational toast with an optional "Leave Anyway" action instead of silently aborting the query.
- **Consumer search isolation**: Consumer HTTP requests no longer appear in Creator's FHIR Request Inspector panel — the two modules now maintain completely separate logging contexts.

### Technical changes

- `src/renderer/features/about/AboutPage.tsx`: adds a "Remind Me Later" ghost button (neutral color, positioned between the two existing buttons) whose `onClick` resets local update state without calling `skipUpdateVersion`.
- `src/renderer/App.tsx`: inserts a no-op `remindLater` action into `showUpdateToast`'s actions array, maintaining the same three-button order.
- `src/renderer/services/fhirClient.ts`: adds Composition chain search (Step 4), replaces `performLoggedRequest` inside `searchBundles` with a plain `fetch` wrapper that carries `AbortSignal` but skips inspector logging, plumbs `signal` through all internal requests.
- `src/renderer/store/consumerSearchStore.ts`: new Zustand store tracking `isSearching` state shared between `SearchForm` and `ConsumerPage`.
- `src/renderer/hooks/useGuardedNavigate.ts`: reads `isSearching` from store and shows a toast with a "Leave Anyway" action instead of blocking navigation.
- `src/renderer/features/consumer/SearchForm.tsx`: adds `AbortController` ref, cancel button UI, `AbortError` handling, `onBusyChange` / `onSearchStart` callbacks.
- `src/renderer/features/consumer/ResultList.tsx`: adds loading skeleton state and cancelled state display.
- i18n: adds `about.update.remindLater` key in both `en` and `zh-TW` settings namespaces; adds `search.cancelButton`, `results.cancelled`, `results.cancelledHint`, `results.searching`, `results.searchingHint`, `page.searchingGuard.*` keys in both consumer namespaces.
