# RxFHIR v1.0.26

## Added

### Consumer result sort and keyword filter

Once a search returns results, a sort dropdown and keyword filter input appear in the results header:

**Sort options (client-side, no re-query needed):**
- Newest First *(default)* — by `Composition.date`, descending
- Oldest First — by `Composition.date`, ascending
- Name (A → Z) — by patient name, alphabetical
- Name (Z → A) — by patient name, reverse alphabetical

Sort preference is preserved across queries; switching to a new search does not reset it.

**Keyword filter:**
- Matches simultaneously against patient name, patient identifier, and organization name
- Case-insensitive, instant (no debounce needed at these result sizes)
- When active and narrowing the set, a `N match ×` badge appears in the header — click it or the `×` inside the input to clear
- Resets automatically when a new search starts
- If the filter hides all results, a "No results match the filter" empty state is shown with a "Clear filter" button
- Keyboard navigation (↑ ↓ Home End) operates on the filtered + sorted array

## Fixed

### Sidebar logo clipped on Windows and Linux

The `℞` logo in the top-left of the sidebar was visually clipped or incorrectly offset on Windows and Linux builds because a 28 px macOS traffic-light drag zone (`titlebar-drag-region`) was always rendered regardless of platform.

- `Sidebar.tsx`: the `titlebar-drag-region` div is now rendered **only on macOS** (detected via `navigator.platform` / `navigator.userAgent`); on Windows and Linux it is omitted entirely
- `App.tsx`: the utility controls row (Activity Center, Feature Showcase, theme toggle, language toggle) similarly applies the macOS drag-region class only on macOS; on other platforms it uses a plain `h-9` toolbar row so all controls remain visible with correct alignment

### Technical changes

- `src/renderer/features/consumer/ResultList.tsx`: adds `sortKey` and `filterText` local state, `sortedAndFiltered` useMemo, updated header bar, filter no-match empty state, keyboard navigation targeting filtered array
- `src/renderer/components/Sidebar.tsx`: conditionally renders `titlebar-drag-region`
- `src/renderer/App.tsx`: conditionally applies `titlebar-spacer` vs plain `h-9`
- i18n: adds `results.sort*` and `results.filter*` keys to both `en` and `zh-TW` consumer namespaces
