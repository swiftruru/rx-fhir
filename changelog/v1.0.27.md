# RxFHIR v1.0.27

## Added

### Side-by-side Bundle Diff

After selecting a prescription in Consumer, every other result card shows a labeled **⊟ 比較 / Compare** button (outline style with icon + text). Clicking it opens a full-screen diff dialog comparing two bundles field by field.

**Diff dialog features:**
- All structured fields are aligned side by side: Prescription info, Patient, Practitioner, Organization, Encounter, Condition, Observation, Coverage, and Medication & MedicationRequest
- Fields that differ are highlighted with an amber background row; identical fields remain neutral
- A badge in the dialog header shows the total number of differing fields (or "No differences found")
- **Swap A / B** button flips the comparison direction without closing the dialog
- Full-height scrollable content with sticky header; close button returns to the original detail view without changing the selected bundle

**Compare button discoverability:**
- Changed from an icon-only ghost button to an outline button with icon + short text label ("比較" / "Compare")
- The outline border is visible without hovering, making the action immediately apparent to first-time users
- Button only appears when a bundle is selected and the card is not the currently selected one

**Feature Showcase integration:**
- A new `consumerBundleDiff` step (step 13 of 17) spotlights the results panel with the diff dialog open live during the coaching panel
- Coaching panel placed at `bottom-right` to avoid obscuring the prescription title columns in the dialog

### Result CSV Export

A download button appears in the results toolbar once results are loaded. Clicking it exports the current filtered and sorted list as a CSV file.

- **RFC 4180 compliant** with UTF-8 BOM so Chinese characters display correctly in Excel and Numbers without manual encoding selection
- Exports only the currently visible (filtered + sorted) results, not the full server result set
- Eight columns: Bundle ID, Patient Name, Patient Identifier, Date, Organization, Conditions, Medications, Source
- Filename includes the export date: `rxfhir-results-YYYY-MM-DD.csv`
- Shows a spinner during export and a success toast with the saved file path

## Fixed

### Feature Showcase no longer pollutes submission history

Running the Feature Showcase previously injected mock submission records ("王小明", "潘小如") into `historyStore`, causing them to appear in the Consumer **History** tab's submission list alongside the user's real records. After showcase exit the records were supposed to be restored, but several failure modes could leave mock data permanently in localStorage.

- `FeatureShowcaseRunner` no longer calls `useHistoryStore.setState()` at all — the submission history store is completely untouched by the showcase
- `ShowcaseBackup` interface and all backup / restore paths for `historyStore` have been removed

### Replay no longer overwrites the real-data backup

When the user clicked **Replay** after a showcase completed, `runId` incremented and triggered the backup effect again. At that point `historyStore` already held mock data, so the backup was overwritten with mock records. On final exit, restore would write mock data back — permanently losing the user's real submissions.

- The backup is now guarded by `if (!backupRef.current)` so it is taken **only once** per showcase session; Replay reuses the original pre-showcase backup

### Showcase restore now fires on the `completed` status

The restore effect previously triggered only when `status === 'idle'`. If the app was quit after the tour completed naturally but before the user clicked **Close** (which transitions to `idle`), localStorage would retain mock store state across restarts.

- Restore now fires on both `'idle'` and `'completed'`, so localStorage is written back to real user data as soon as the last step finishes

### Consumer FHIR requests no longer appear in Creator's inspector

All requests made from the Consumer page (bundle searches, bundle GETs) were written to the shared `fhirInspectorStore` and appeared in the Creator's right-panel FHIR Request Inspector, mixed in with Creator's own POST/PUT/GET traffic.

- `fhirClient.ts` reads `window.location.hash` at request time and tags each `FhirRequestEntry` with its originating module (`creator`, `consumer`, `settings`, or `app`)
- `ResourceStepper.tsx` filters the store history to entries where `module === 'creator'` (or legacy entries without a module tag); `latestRequest` is derived from the filtered list
- Consumer requests are stored in the shared log but never rendered in the Creator panel

## Technical changes

- `src/renderer/lib/bundleDiff.ts` *(new)*: pure diff utility — `diffBundles(a, b): BundleDiffResult`; extracts fields from both bundles using the same `getResource<T>(entries, type)` pattern as `PrescriptionDetail`; no external diff library
- `src/renderer/features/consumer/BundleDiffDialog.tsx` *(new)*: diff dialog component; `swapped` state controls A/B order; `DiffRow` sub-component renders a 3-column grid (label | A value | B value) with amber highlight when `isDifferent`
- `src/renderer/features/consumer/ConsumerPage.tsx`: adds `diffTarget` state, `handleCompare()` handler, `BundleDiffDialog` render, and showcase `useEffect` logic for `bundleDiffTargetId`
- `src/renderer/features/consumer/ResultList.tsx`: adds `onCompare` prop, Compare outline button per card, CSV export button + `handleExportCsv()` async handler, `isExporting` state
- `src/renderer/services/bundleFileService.ts`: adds `csvEscapeField()` helper and `exportResultsCsv(results)` function
- `src/renderer/store/fhirInspectorStore.ts`: adds `module?: string` field to `FhirRequestEntry`; `startRequest()` accepts optional `module` parameter
- `src/renderer/services/fhirClient.ts`: adds `getCurrentModule()` reading `window.location.hash`; passes module to every `startRequest()` call
- `src/renderer/features/creator/ResourceStepper.tsx`: filters `fhirInspectorStore` history to creator-scoped entries; derives `latestRequest` from filtered list
- `src/renderer/components/FeatureShowcaseRunner.tsx`: removes all `historyStore` backup/restore; adds `if (!backupRef.current)` guard on backup; restore fires on both `idle` and `completed`
- `src/renderer/components/FeatureShowcaseCoach.tsx`: raises z-index to `z-[60]` so coaching panel renders above `Dialog` overlays (`z-50`)
- `src/renderer/showcase/types.ts`: adds `'consumerBundleDiff'` to `FeatureShowcaseStepId`; adds `bundleDiffTargetId?: string` to `FeatureShowcaseConsumerOverride`
- `src/renderer/showcase/featureShowcaseScript.ts`: adds `consumerBundleDiff` step (index 12, between `consumerExport` and `settingsServer`)
- i18n: adds `results.compareButton`, `results.compareShortLabel`, `results.exportCsvButton`, `results.exportCsvSuccess`, `results.exportCsvFileName`, `results.exportCsvPartialHint`, `results.diffDialog.*` keys to both `en` and `zh-TW` consumer namespaces; adds `steps.consumerBundleDiff` to both showcase namespaces
