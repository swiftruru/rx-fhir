# RxFHIR v1.0.29

## Fixed

### macOS app icon now displays correctly in the About window

The app icon was showing a generic document placeholder in the custom About
window on packaged macOS builds. Two separate issues caused this:

**Non-square source PNG**

`build/icon.png` was 861×859 pixels (not square). electron-builder requires
a square image to generate a valid `.icns` file; a non-square source produces
a broken or empty icon set, which is what macOS rendered in the About window
and in the Dock.

- `build/icon.png` replaced with a 1024×1024 square version (resized from the
  original; the 2-pixel asymmetry was imperceptible)
- A proper `build/icon.icns` generated via `iconutil` from a complete iconset
  covering all required sizes (16, 32, 128, 256, 512, 1024 px and @2x variants)
- `electron-builder.yml`: `mac.icon` changed from `build/icon.png` to
  `build/icon.icns` so the app bundle receives a correctly structured icon file

**Wrong production icon path in main process**

`getIconPath()` used `resolveMainPath('../../resources/icon.png')` in
production, which resolves relative to `import.meta.url` inside the asar
archive. This path pointed to a location inside the asar that does not exist,
causing `nativeImage.createFromPath()` to silently return an empty image.

- `getIconPath()` now uses `join(app.getAppPath(), '..')` to locate the
  platform Resources directory reliably at runtime
- On macOS it returns `icon.icns` (placed there by electron-builder as part of
  the standard macOS app bundle structure)
- On Windows and Linux it returns `icon.png` (placed there via `extraResources`)
- `electron-builder.yml`: added `extraResources` entry to copy `build/icon.png`
  to `icon.png` in the packaged app's Resources directory for Windows and Linux

## Technical changes

- `build/icon.png` — resized to 1024×1024 square
- `build/icon.icns` — new file, generated with `iconutil` from a full iconset
- `electron-builder.yml` — `mac.icon: build/icon.icns`; added `extraResources`
  block copying `build/icon.png → icon.png`
- `src/main/index.ts` — `getIconPath()` uses `app.getAppPath()` in production
  instead of `resolveMainPath` with a relative asar-internal path
