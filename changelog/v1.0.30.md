# RxFHIR v1.0.30

## Fixed

### macOS About window icon rendering in packaged builds

The packaged macOS app could still render a broken icon in the custom About
window even after the bundle icon assets were corrected. The remaining issue
was that runtime `nativeImage` consumers in the main process were loading the
packaged `.icns` file directly.

- `src/main/index.ts` now uses the packaged `icon.png` for runtime icon loading
  in the custom About window, `BrowserWindow`, and the macOS Dock icon
- `electron-builder.yml` still keeps `build/icon.icns` as the bundle icon so
  Finder, app metadata, and release artifacts continue to use the standard
  macOS icon format
- verified locally by launching the packaged `dist/mac-arm64/RxFHIR.app` and
  confirming that `About RxFHIR` displays the correct icon
