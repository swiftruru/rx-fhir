# RxFHIR v1.0.31

## Fixed

### Live Demo stop and restart no longer trap sidebar navigation

Stopping Live Demo in the middle of a run and then pressing `重新開始` could
leave stale demo tasks alive in the background. Those old async fill and submit
flows could still write Creator state back after reset, which made sidebar
navigation to Consumer, Settings, and About flash and then stay on Creator.

- `src/renderer/store/liveDemoStore.ts` now advances the demo `runId` on
  `stop()` and `dismiss()` so an interrupted demo run becomes invalid
  immediately
- `src/renderer/components/LiveDemoRunner.tsx` now re-checks abort state after
  fill and submit phases, and only starts a demo run when the store is actually
  in `running` state
- `src/renderer/lib/liveDemoRuntime.ts`,
  `src/renderer/hooks/useLiveDemoFormController.ts`, and
  `src/renderer/hooks/useCreatorMockFill.ts` now make demo typing, reveal, and
  submit flows stop-aware so paused or cancelled runs cannot continue
- `src/renderer/features/creator/ResourceStepper.tsx` and
  `src/renderer/features/creator/forms/CompositionForm.tsx` now ignore stale
  submit results from cancelled demo runs
- `src/renderer/hooks/useGuardedNavigate.ts` now blocks navigation only when
  leaving `/consumer` during an active search, instead of treating unrelated
  pages as guarded
- `src/renderer/App.tsx` clears the global Consumer searching flag when the app
  is no longer on `/consumer`, preventing a stale search state from leaking into
  other pages
- `src/renderer/features/creator/CreatorPage.tsx` now clears the FHIR request
  inspector when `重新開始` is confirmed, so reset returns Creator to a clean
  state

## Verified

- `npm run typecheck`
- local Electron repro of `開始 Live Demo -> 自動播放 -> 停止 -> 重新開始 ->
  確定清除 -> Settings / About / Consumer`, confirming the app no longer jumps
  back to Creator
