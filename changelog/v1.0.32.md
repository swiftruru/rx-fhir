# RxFHIR v1.0.32

## Fixed

### Windows app window can now close normally

On Windows, the app could become impossible to close after opening Creator with
unsaved draft content. The renderer was registering a `beforeunload` handler
that always called `preventDefault()`, which blocked Electron's normal window
close flow and made the app appear stuck even though draft autosave was already
enabled.

- `src/renderer/App.tsx` no longer installs a global `beforeunload` blocker for
  incomplete Creator drafts
- closing the main window on Windows now falls through to Electron's normal
  `window-all-closed -> app.quit()` path
- draft autosave behavior remains unchanged, so closing the app no longer
  trades off with preserving unfinished Creator work
