# RxFHIR v1.0.33

## Fixed

### Stopping Feature Showcase no longer reapplies demo data after locale changes

Stopping Feature Showcase and then switching the app locale could re-run the
showcase initialization effect because `runId` remained non-zero after stop.
That rebuilt the showcase snapshot and overwrote the real Creator workspace
with fully populated demo data.

- `src/renderer/components/FeatureShowcaseRunner.tsx` now rebuilds showcase
  state only while the showcase is actually `running` or `paused`
- stopping the showcase now preserves the user's real Creator workspace when
  locale or server settings change after the tour ends
- `README.md` now documents that stopping the showcase restores the real
  workspace immediately

### Creator and Consumer page headers now use the same typography scale

The Creator and Consumer top-level workflows were using different title and
subtitle sizes, which made the main page headers feel visually inconsistent.

- `src/renderer/features/consumer/ConsumerPage.tsx` now matches Creator's page
  title and subtitle typography
