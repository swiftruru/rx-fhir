# RxFHIR v1.0.35

## Fixed

### GitHub release builds now install dependencies successfully again

The `v1.0.34` tag exposed a CI-only dependency issue: the newly added test
toolchain pulled in `vitest@4`, which resolved to a `vite@8` branch expecting
`esbuild@^0.27 || ^0.28`, while the application toolchain still targets
`vite@5` and `esbuild@0.21.x`. GitHub Actions rejected the lockfile during
`npm ci` and the release build failed before artifact generation.

- `vitest` is now pinned to the `2.1.9` line, which depends on `vite@^5`
- `package-lock.json` is now back in sync with the repo's Vite 5 toolchain
- local `npm ci` now succeeds again, matching the GitHub Actions install step
