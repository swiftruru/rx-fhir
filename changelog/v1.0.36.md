# RxFHIR v1.0.36

## Added

### Electron UI automation is now a first-class project capability

RxFHIR now includes a Playwright-based Electron UI automation layer that can
launch the real desktop app, drive real renderer UI flows, and validate stable
user-facing outcomes with isolated app state.

- added `npm run test:ui`, `npm run test:ui:headed`, and `npm run test:ui:report`
- added Playwright Electron coverage for About, App shell, Creator, Consumer,
  History, and Settings flows
- added deterministic fixtures, mocked FHIR helpers, persisted-state seeding,
  and Electron launch helpers for stable local runs

## Changed

### Demo and delivery documentation now treat UI automation as a stable v1 scope

The project documentation now explains not only how to run the UI suite, but
also what is intentionally in scope, what is intentionally out of scope, and
how to present the result in a classroom or thesis-demo setting.

- documented the current UI automation delivery boundary in `README.md` and
  `docs/testing/ui-automation.md`
- added a teacher-facing presentation script in
  `docs/testing/ui-automation-demo-script.md`
- wired a GitHub Actions workflow to run typecheck, Vitest, and the Electron UI
  suite and upload Playwright artifacts
