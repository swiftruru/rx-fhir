# RxFHIR v1.0.37

## Added

### Electron UI automation now covers more real product branches

The Playwright-based Electron suite now reaches beyond the original happy-path
flows and verifies additional user-visible branches across About, Consumer,
Creator, History, and Settings.

- added Consumer coverage for date search, complex author search, cancelled
  search state, and load-more pagination
- added Settings coverage for FHIR server test / save / restore
- added Creator coverage for successful bundle submission and Consumer handoff
- added History maintenance coverage for deleting and clearing recent records
  and saved searches
- added About coverage for cached update state, failed update checks, and
  "Remind Me Later" behavior

## Changed

### README now reflects the current UI automation scope more accurately

The README's testing summary now matches the repo's current Electron UI
automation coverage and quality signals.

- updated the UI automation coverage bullets to include the newer About,
  Creator, Consumer, History, and Settings flows
- updated the repository quality summary to mention the local Playwright suite
  and the dedicated GitHub Actions UI automation workflow
