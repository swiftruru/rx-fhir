# RxFHIR v1.0.38

## Added

### Creator-to-Consumer preview is now a first-class local review flow

The final Creator step can now open the assembled document Bundle directly in
Consumer without writing to the server first, making classroom demos and
pre-submit checks safer and easier to explain.

- added a clearly labeled `Consumer Preview` handoff from the final Creator step
- added a prominent `Return to Creator` action while previewing in Consumer
- added preview-session reset behavior so leaving Consumer returns the normal
  query workspace instead of preserving stale audit state

### Consumer now audits every Bundle source with clearer server guidance

Bundle detail now shows the same local-first audit workflow for server search
results, imported files, and Creator previews, then upgrades to hybrid server
`$validate` feedback when the current server supports it.

- added local-first audit cards for server, imported, and preview bundles
- added localized plain-language summaries for common server validation messages
- added validator-environment explanation cards so public HAPI profile and
  terminology limits are separated from true structural bundle errors

### Desktop bundle handoff and persistence now behave more like a real app

Consumer now remembers more of the user's working state and can open bundles
from the desktop environment directly.

- added Consumer workspace persistence for latest search tab, query values,
  selected bundle, and recent imported file
- added OS-level bundle-open handoff for startup arguments and macOS recent
  documents
- added UI automation coverage for preview reset, workspace restore, and
  startup file-open flows

## Changed

### TW EMR preview assembly is now stricter and more server-compatible

The assembled preview bundle now stays closer to the expected Taiwan EMR
document structure, reducing false validation failures on stricter servers.

- aligned Composition section assembly with the EMR document scope
- normalized preview dateTime values before server validation
- updated EMR profile expectations to use `Bundle-EP`,
  `Observation-EP-BodyWeight`, and `Coverage-EMR`
- excluded standalone supplemental `Basic` extension resources from the
  document bundle so they no longer appear as unreachable entries

### Consumer preview and detail layouts now handle narrow windows more reliably

The Consumer preview experience was refined so audit detail remains visible and
scrollable on smaller desktop window sizes.

- fixed preview detail placement on narrow widths so the audit pane moves below
  the search panel when needed
- fixed nested scrolling so the page can still reach the lower audit/detail
  content in compact windows
- fixed audit card overflow caused by long server diagnostics

### Documentation now reflects the current release surface more accurately

- updated the README feature summary for Consumer Preview, hybrid audit, server
  capability diagnostics, workspace persistence, and desktop bundle handoff
- refreshed the README quality section to match the current Electron UI
  automation count
- expanded UI automation docs and demo notes for the newer preview and
  validation flows
