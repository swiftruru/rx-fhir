# RxFHIR v1.0.39

## Fixed

### Document Bundle submission no longer fails with HAPI-2840 duplicate error

When submitting a prescription through the Creator wizard, the Composition
resource was being POSTed to the FHIR server individually before being
included again inside the Document Bundle. HAPI FHIR detected the duplicate
Composition and rejected the bundle with error HAPI-2840.

- removed the separate Composition POST; the Composition is now created
  exclusively through the Document Bundle submission
- replaced server-assigned IDs with bundle-scoped UUIDs in all bundle entry
  resources, preventing duplicate detection for Patient, Organization, and
  other resources that were created individually in earlier steps
- extracted the Composition ID from the bundle response after successful
  submission so the UI continues to display the correct server-assigned ID

This fix applies to all prescription templates (foundation, acute, chronic,
pediatric), not just the specific scenario that originally triggered the error.
