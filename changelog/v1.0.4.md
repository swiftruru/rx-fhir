# RxFHIR v1.0.4

## Changed

- Updated the Creator JSON preview to switch between light and dark visual themes so it stays consistent with the active app theme.

## Fixed

- Fixed duplicate `Encounter` creation on public HAPI servers by reusing existing resources that match the submitted visit context instead of sending a duplicate `POST`.
- Fixed duplicate `Condition` creation on public HAPI servers by reusing existing resources that match the submitted diagnosis context instead of sending a duplicate `POST`.
- Fixed duplicate `Observation` creation on public HAPI servers by reusing existing resources that match the submitted observation context instead of sending a duplicate `POST`.
- Fixed duplicate `MedicationRequest` creation on public HAPI servers by reusing existing resources that match the submitted prescription context instead of sending a duplicate `POST`.
- Fixed duplicate `Basic` extension creation on public HAPI servers by reusing existing resources that match the submitted supplemental data context instead of sending a duplicate `POST`.
