# RxFHIR v1.0.40

## Fixed

### Document Bundle submission no longer fails with HAPI-2840 or HAPI-0505

When submitting a prescription through the Creator wizard, two HAPI FHIR
server errors could occur:

**HAPI-2840 (duplicate Composition)**: The Composition was POSTed
individually before being included in the Document Bundle. HAPI detected the
duplicate and rejected the bundle.

- removed the separate Composition POST; the Composition is now created
  exclusively through the Document Bundle submission
- extracted the Composition ID from the bundle response so the UI continues
  to display the correct server-assigned ID

**HAPI-0505 (invalid reference format)**: Internal references were rewritten
from `ResourceType/id` to `urn:uuid:` format, which HAPI rejects in
Document Bundles with "Does not contain resource type".

- Document Bundles now keep the original `ResourceType/id` references
  instead of rewriting them to `urn:uuid:` format
- removed unused `buildReferenceLookup` and related dead code
- `fullUrl` entries are still generated as `urn:uuid:` for bundle-level
  identification, which is unaffected by this change

Both fixes apply to all prescription templates (foundation, acute, chronic,
pediatric).

### Error toast now stays visible long enough to read

Error toasts previously auto-dismissed after 4.2 seconds, which was too
short for FHIR error messages. Error toasts now default to 10 seconds.
