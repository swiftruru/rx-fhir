# RxFHIR v1.0.5

## Added

- Added human-friendly FHIR `OperationOutcome` error messages with expandable raw error details for troubleshooting.
- Added local Creator draft auto-save and automatic draft restoration on the next app launch.
- Added a structured prescription summary review card before final bundle submission.
- Added one-click handoff from Creator to Consumer, including automatic query prefill, auto-search, and selection of the newly created bundle.

## Changed

- Updated Creator success feedback so completed steps keep showing their latest success or reuse message after you leave and revisit the step.
- Updated duplicate-protection UX to explicitly show when an existing server-side resource is reused instead of presenting only a generic success state.
- Updated README to reflect the current Creator draft flow, submission review flow, Consumer handoff behavior, and improved FHIR error handling.

## Fixed

- Fixed step-level success alerts disappearing after navigating away from a completed Creator step and returning later.
- Reduced ambiguity around server-side resource reuse by preserving clear success feedback for Coverage, Medication, Encounter, and other deduplicated resources.
