# RxFHIR v1.0.6

## Changed

- Updated the GitHub Actions release workflow to read release notes from the changelog file stored in the tagged commit, preventing false "missing changelog" failures when publishing releases.
- Upgraded the artifact upload and download actions used by the release pipeline to newer major versions to stay aligned with GitHub's Node.js runtime migration.

## Fixed

- Fixed the release pipeline so tagged builds publish GitHub Releases from the correct tagged source instead of depending on the default checkout state.
- Fixed the `v1.0.5` release issue path by moving forward with a clean patch release process for subsequent tags.
