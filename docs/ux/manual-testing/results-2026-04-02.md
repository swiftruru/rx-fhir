## 風險與後續建議

1. `System Events` 現在可用於鍵盤操作與截圖輔助，但直接列舉 Electron 視窗的可存取元素仍不穩，後續桌面 UX 回歸仍建議以 `CDP + screenshot + keyboard automation` 為主。
2. 若要持續做這類 Electron UX 回歸，建議把這次用到的 CDP 驗證腳本正式整理成 repo 內工具，而不是只留在 `/tmp`。
3. 若後續要驗更多原生互動，可補：
   - `openExternalUrl()` 是否正確交給系統瀏覽器
   - recent files / recent bundles 與 command palette 的交互回歸
   - menu bar / dock / About window 的桌面體驗驗收
